## 整体架构

confg 项目分为 **两条独立的数据流**：

### 流程 A: 运行时配置查询（runtime）

```
YAML/JSON 文件
    ↓
parser.ConfigNode (临时中间表示)
    ↓
tree.ConfigTree (Trie 树结构)
    ↓
运行时动态查询 (Get/Set/Watch)
```

**用途**: 运行时从多个配置源（文件、环境变量、etcd 等）加载配置，支持优先级合并和热更新。

---

### 流程 B: 代码生成（本项目核心功能）

```
YAML/JSON 文件
    ↓
parser.ConfigNode (临时中间表示)
    ↓
analyzer.AnalyzeResult (Go 类型定义)
    ↓
codegen (代码生成器)
    ↓
Go 源码文件
```

**用途**: 根据配置文件自动生成强类型的 Go struct，提供编译时类型检查。

---

## 数据流转

### 步骤 1: YAML → Parser

**输入**: YAML 文本
```yaml
server:
  host: "localhost"
  port: 8080
```

**处理**: `parser.YAMLParser`
- 使用 `gopkg.in/yaml.v3` 解析 YAML
- 递归构建 `ConfigNode` 树

**输出**: `parser.ConfigNode` 树
```go
ConfigNode {
    Key: "root",
    Type: TypeObject,
    Children: {
        "server": ConfigNode {
            Key: "server",
            Type: TypeObject,
            Children: {
                "host": ConfigNode {
                    Key: "host",
                    Type: TypeString,
                    Value: "localhost"
                },
                "port": ConfigNode {
                    Key: "port",
                    Type: TypeInt,
                    Value: 8080
                }
            }
        }
    }
}
```

---

### 步骤 2: ConfigNode → Analyzer

**输入**: `parser.ConfigNode` 树

**处理**: `analyzer.Analyzer`
- 遍历 ConfigNode 树
- 推断每个节点的 Go 类型
- 生成 struct 定义

**输出**: `analyzer.AnalyzeResult`
```go
AnalyzeResult {
    RootStruct: &StructDef {
        Name: "Config",
        Fields: [
            {
                Name: "Server",
                Type: "ServerConfig",
                JSONTag: "server",
                YAMLTag: "server"
            }
        ]
    },
    SubStructs: {
        "ServerConfig": &StructDef {
            Name: "ServerConfig",
            Fields: [
                {Name: "Host", Type: "string", JSONTag: "host"},
                {Name: "Port", Type: "int", JSONTag: "port"}
            ]
        }
    }
}
```

---

### 步骤 3: AnalyzeResult → CodeGen

**输入**: `analyzer.AnalyzeResult`

**处理**: `codegen.Generator` (待实现)
- 将 StructDef 转换为 Go 源码
- 添加 tags (json, yaml, mapstructure)
- 格式化代码

**输出**: Go 源码
```go
// Code generated by confg. DO NOT EDIT.
package main

type Config struct {
    Server ServerConfig `json:"server" yaml:"server" mapstructure:"server"`
}

type ServerConfig struct {
    Host string `json:"host" yaml:"host" mapstructure:"host"`
    Port int    `json:"port" yaml:"port" mapstructure:"port"`
}
```

## 核心模块

### 1. Parser 模块 (`internal/parser`)

**职责**: 解析配置文件，生成临时的树结构

**核心文件**:
- `parser.go` - 解析器接口定义
- `yaml.go` - YAML 解析实现
- `json.go` - JSON 解析实现
- `types.go` - ConfigNode 数据结构
- `factory.go` - 根据文件扩展名自动选择解析器
- `bridge.go` - 将 ConfigNode 转换为 tree.ConfigTree (用于运行时)

**关键数据结构**:
```go
type ConfigNode struct {
    Key      string                 // 字段名
    Value    any                    // 实际值
    Type     ValueType              // 类型 (String/Int/Object/Array 等)
    Children map[string]*ConfigNode // 子节点 (Object 类型)
    Items    []*ConfigNode          // 数组元素 (Array 类型)
}
```

**设计思想**:
- **为什么需要 ConfigNode？** YAML 和 JSON 的原始结构是 `map[string]interface{}` 和 `[]interface{}`，难以统一处理。ConfigNode 提供了统一的树形接口。
- **为什么是临时的？** Parser 只负责解析，不关心最终用途。下游模块（analyzer 或 tree）根据需要转换。

---

### 2. Tree 模块 (`internal/tree`)

**职责**: 提供运行时的 Trie 树结构，支持多源配置合并

**核心文件**:
- `tree.go` - ConfigTree 实现
- `node.go` - TrieNode 实现
- `types.go` - 类型定义和优先级配置

**关键特性**:
- **Trie 结构**: `server.database.host` → 拆分为路径节点，O(k) 查询
- **多源合并**: 支持文件、环境变量、远程配置，按优先级覆盖
- **优先级**: CodeOverride(100) > SystemEnv(40) > SessionEnv(30) > File(20) > Remote(10) > Default(0)

**与 Parser 的关系**:
- Parser 生成 ConfigNode（临时）
- `bridge.go` 将 ConfigNode 转换为 ConfigTree（持久）
- ConfigTree 用于运行时查询，ConfigNode 用于代码生成

---

### 3. Analyzer 模块 (`internal/analyzer`)

**职责**: 从 ConfigNode 推断 Go 类型结构

**核心文件**:
- `analyzer.go` - 核心推断逻辑
- `types.go` - StructDef/FieldDef 数据结构
- `naming.go` - 命名转换规则

**核心逻辑** (详见下一节):
1. 对象节点 → struct
2. 数组节点 → []T
3. 基础类型 → string/int/bool/float64
4. 命名转换: snake_case → PascalCase

## Analyzer 原理详解

### 1. 核心数据结构

#### StructDef - struct 定义
```go
type StructDef struct {
    Name   string      // struct 名称 (如 "ServerConfig")
    Fields []*FieldDef // 字段列表
}
```

#### FieldDef - 字段定义
```go
type FieldDef struct {
    Name         string // 字段名 (PascalCase, 如 "MaxConnections")
    Type         string // Go 类型 (如 "string", "[]int", "*DatabaseConfig")
    JSONTag      string // JSON tag (原始 key, 如 "max_connections")
    YAMLTag      string // YAML tag
    MapStructTag string // mapstructure tag
    Comment      string // 字段注释
}
```

#### AnalyzeResult - 分析结果
```go
type AnalyzeResult struct {
    RootStruct *StructDef            // 根配置 struct (Config)
    SubStructs map[string]*StructDef // 所有嵌套 struct
}
```

---

### 2. 类型推断算法

#### 算法流程

```
Analyze(root *ConfigNode) → AnalyzeResult
    ↓
analyzeObject(structName, node) → StructDef
    ↓ (遍历每个子节点)
analyzeField(key, childNode) → FieldDef
    ↓ (根据节点类型分支)
    ├─ Primitive → GoType(node.Type)
    ├─ Array → analyzeArrayType(...)
    └─ Object → 递归 analyzeObject(...)
```

#### 示例 1: 基础类型推断

**输入**:
```yaml
host: "localhost"
port: 8080
enabled: true
```

**ConfigNode**:
```go
ConfigNode {
    Type: TypeObject,
    Children: {
        "host": {Type: TypeString, Value: "localhost"},
        "port": {Type: TypeInt, Value: 8080},
        "enabled": {Type: TypeBool, Value: true}
    }
}
```

**推断逻辑**:
```go
// analyzeField("host", node)
field := &FieldDef{
    Name:    ToFieldName("host"),      // → "Host"
    Type:    GoType(TypeString),        // → "string"
    JSONTag: "host",
    YAMLTag: "host"
}
```

**输出**:
```go
type Config struct {
    Host    string `json:"host" yaml:"host"`
    Port    int    `json:"port" yaml:"port"`
    Enabled bool   `json:"enabled" yaml:"enabled"`
}
```

---

#### 示例 2: 嵌套对象

**输入**:
```yaml
server:
  host: "localhost"
  port: 8080
```

**ConfigNode**:
```go
ConfigNode {
    Type: TypeObject,
    Children: {
        "server": ConfigNode {
            Type: TypeObject,
            Children: {
                "host": {Type: TypeString, Value: "localhost"},
                "port": {Type: TypeInt, Value: 8080}
            }
        }
    }
}
```

**推断逻辑**:
```go
// 步骤 1: 分析根节点
analyzeObject("Config", root)
    ↓
// 步骤 2: 遇到 "server" 字段，是 Object 类型
analyzeField("server", serverNode)
    → 发现是 Object，递归创建新 struct
    ↓
// 步骤 3: 创建 ServerConfig struct
nestedStructName := ToStructName("server")  // → "ServerConfig"
nestedStruct := analyzeObject("ServerConfig", serverNode)
result.AddStruct(nestedStruct)

// 步骤 4: 字段类型指向嵌套 struct
field.Type = "ServerConfig"
```

**输出**:
```go
type Config struct {
    Server ServerConfig `json:"server" yaml:"server"`
}

type ServerConfig struct {
    Host string `json:"host" yaml:"host"`
    Port int    `json:"port" yaml:"port"`
}
```

---

#### 示例 3: 数组类型

**输入**:
```yaml
tags:
  - "production"
  - "web"
```

**ConfigNode**:
```go
ConfigNode {
    Type: TypeObject,
    Children: {
        "tags": ConfigNode {
            Type: TypeArray,
            Items: [
                {Type: TypeString, Value: "production"},
                {Type: TypeString, Value: "web"}

        }
    }
}
```

**推断逻辑**:
```go
// analyzeArrayType("tags", arrayNode)
firstItem := node.Items[0]  // 取第一个元素推断类型

    return "[]" + elemType              // → "[]string"
}
```

**输出**:
```go
type Config struct {
    Tags []string `json:"tags" yaml:"tags"`
}
```

#### 示例 4: 对象数组

**输入**:
```yaml
servers:
  - host: "server1"
    port: 8080
  - host: "server2"
    port: 8081
```

**ConfigNode**:
```go
ConfigNode {
    Type: TypeObject,
    Children: {
        "servers": ConfigNode {
            Type: TypeArray,
            Items: [
                ConfigNode {
                    Type: TypeObject,
                    Children: {
                        "host": {Type: TypeString, Value: "server1"},
                        "port": {Type: TypeInt, Value: 8080}
                    }
                },
                ...
            ]
        }
    }
}
```

**推断逻辑**:
```go
// analyzeArrayType("servers", arrayNode)
firstItem := node.Items[0]

if firstItem.IsObject() {
    // 数组元素是对象，创建 struct
    itemStructName := ToStructName("servers")  // → "ServersConfig"
    
    // 去掉复数 's' (简单处理)
    // "ServersConfig" → "ServerConfig"
    
    itemStruct := analyzeObject("ServerConfig", firstItem)
    result.AddStruct(itemStruct)
    
    return "[]ServerConfig"
}
```

**输出**:
```go
type Config struct {
    Servers []ServerConfig `json:"servers" yaml:"servers"`
}

type ServerConfig struct {
    Host string `json:"host" yaml:"host"`
    Port int    `json:"port" yaml:"port"`
}
```

### 3. 命名转换规则

#### snake_case → PascalCase

**规则**:
1. 按 `_`, `-`, 空格分割单词
2. 每个单词首字母大写
3. 常见缩写词全大写 (API, ID, URL, HTTP 等)

**示例**:
```go
ToFieldName("max_connections")  → "MaxConnections"
ToFieldName("api_key")          → "APIKey"       // API 全大写
ToFieldName("user_id")          → "UserID"       // ID 全大写
ToFieldName("db_host")          → "DBHost"       // DB 全大写
```

**缩写词列表** (可扩展):
```go
API, URL, URI, HTTP, HTTPS, JSON, XML, HTML,
SQL, DB, TCP, UDP, IP, TLS, SSL, CPU, RAM, UUID, ID
```

#### struct 命名规则

**规则**: 字段名 + "Config" 后缀

**示例**:
```go
ToStructName("server")          → "ServerConfig"
ToStructName("database")        → "DatabaseConfig"
ToStructName("max_connections") → "MaxConnectionsConfig"
ToStructName("root")            → "Config"  // 特殊: 根节点
```

### 4. 核心函数解析

#### `Analyze(root *ConfigNode)` - 入口函数

```go
func (a *Analyzer) Analyze(root *parser.ConfigNode) (*AnalyzeResult, error) {
    // 1. 验证根节点是 Object 类型
    if !root.IsObject() {
        return nil, fmt.Errorf("root must be object")
    }

    // 2. 分析根节点，生成 Config struct
    rootStruct := a.analyzeObject("Config", root)
    a.result.RootStruct = rootStruct

    return a.result, nil
}
```

---

#### `analyzeObject(structName, node)` - 分析对象节点

```go
func (a *Analyzer) analyzeObject(structName string, node *ConfigNode) *StructDef {
    def := &StructDef{
        Name:   structName,
        Fields: make([]*FieldDef, 0, len(node.Children)),
    }

    // 遍历所有子节点
    for key, child := range node.Children {
        field := a.analyzeField(key, child)
        def.Fields = append(def.Fields, field)
    }

    return def
}
```

---

#### `analyzeField(key, node)` - 分析字段节点

```go
func (a *Analyzer) analyzeField(key string, node *ConfigNode) *FieldDef {
    field := &FieldDef{
        Name:         ToFieldName(key),  // 转换命名
        JSONTag:      key,                // 保留原始 key
        YAMLTag:      key,
        MapStructTag: key,
    }

    // 根据节点类型推断字段类型
    switch {
    case node.IsPrimitive():
        // 基础类型: string, int, float64, bool
        field.Type = GoType(node.Type)

    case node.IsArray():
        // 数组类型
        field.Type = a.analyzeArrayType(key, node)

    case node.IsObject():
        // 嵌套对象，创建新的 struct
        nestedStructName := ToStructName(key)
        nestedStruct := a.analyzeObject(nestedStructName, node)
        
        // 添加到结果中
        a.result.AddStruct(nestedStruct)
        
        // 字段类型为嵌套 struct
        field.Type = nestedStructName

    default:
        field.Type = "interface{}"
    }

    return field
}
```

---

#### `analyzeArrayType(key, node)` - 分析数组类型

```go
func (a *Analyzer) analyzeArrayType(key string, node *ConfigNode) string {
    if len(node.Items) == 0 {
        return "[]interface{}"  // 空数组
    }

    // 取第一个元素推断类型
    firstItem := node.Items[0]

    var elemType string
    switch {
    case firstItem.IsPrimitive():
        // 基础类型数组: []string, []int 等
        elemType = GoType(firstItem.Type)

    case firstItem.IsObject():
        // 对象数组: []ServerConfig
        itemStructName := ToStructName(key)
        
        // 去掉复数 's' (简单处理)
        if strings.HasSuffix(itemStructName, "sConfig") {
            itemStructName = itemStructName[:len(itemStructName)-7] + "Config"
        }
        
        itemStruct := a.analyzeObject(itemStructName, firstItem)
        a.result.AddStruct(itemStruct)
        elemType = itemStructName

    case firstItem.IsArray():
        // 嵌套数组: [][]string
        elemType = a.analyzeArrayType(key+"_item", firstItem)

    default:
        elemType = "interface{}"
    }

    return "[]" + elemType
}
```

## 总结

### 架构亮点

1. **清晰的分层**: Parser → Analyzer → CodeGen，职责分明
2. **临时 vs 持久**: ConfigNode (临时) vs ConfigTree (持久)
3. **类型安全**: 编译时类型检查 + 运行时动态查询
4. **可扩展性**: 
   - 新增解析器: 实现 Parser 接口
   - 新增配置源: 扩展 tree.SourceType
   - 自定义命名: 修改 naming.go
